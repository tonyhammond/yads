yads: DObject Model Properties
Properties come in two kinds:
General Properties
These can be applied to both application objects as well as to system objects.
System Properties
These are specific to system objects only. Such objects describe DOI Registration Agents, Application Profiles, etc.
The table below lists and defines each property by kind.

General Properties (8):
"access" [string]
A label to mark off an access point on a resource. Example is "default" to indicate to a DOI Service the default resource to return.
"contains" [string (keyword)]
Property set when a new container with a specified ordering is added to the resource group. Takes as value one of: "alt" (alternatives), "bag" (unordered), "seq" (sequence).
"detail" [string]
A human readble description of a resource.
"label" [string]
A presentational element standing for a resource in a display service.
"resolve" [string (keyword)]
A directive which controls the resolution behaviour. Example is a directive to a DOI Service not to bundle a user-supplied parameter string with the resolution result set. Takes as value one of: "parameters", "no_parameters".
"resource" [uri]
The group resource being qualified. This can be variously another "DObject" (identified by DOI, using the unregistered URI scheme "doi:"), a general web "resource" (identified by any URI), or a "literal", ie an inline "resource" (identified by a URI of scheme "data:").
"role" [string (keyword)]
Indicates the relationship of a DObject resource. Takes as value one of: "alias", "ancestor", "child", "descendant", "following", "following_sibling", "parent", "preceding", "preceding_sibling".
"type" [string]
An extensibility type mechanism. The value is the application type (or property) of a group resource and the definition is maintained in a schema registered under the Application Profile.
System Properties (5):
"agent" [uri (doi:)]
A DOI Registration Agent.
"null" [uri (doi:)]
The positive absence of value. Similar to "nil", "null" or "undef" in programming languages.
"profile" [uri (doi:)]
A DOI Application Profile.
"schema" [uri (doi:)]
An application-specific schema which defines registered types, etc.
"service" [uri (doi:)]
A registered service. Example might be a metadata lookup service.

See Design Principles for further info. The underlying data model is RDF - An RDFS Schema is available.
