YADS Classes & Properties

YADS defines a basic "1+7" properties model for an "Item" object and a "2+7" properties model for a "Nest" object.

The one (mandatory) property on each "Item" is a "resource" or alternatively - as a resource proxy - a "collection" which attributes a set of "Item" objects (with an ordering of "unordered", "sequenced", or "choice") through which a resource hierarchy can be propagated. The two (mandatory) properties on each "Nest" are a "resource" and a "collection" which are bound together..

Each "Item" or "Nest" object may have 0..7 qualifying properties associated with it. Base properties are thus as follows: 


  Props 1 = ("resource" | "collection")                           # mandatory, one of
  Props 2 = ("resource" & "collection")                           # mandatory, two of
  Props 7 = (["access"] & ["detail"] & ["directive"] & ["label"]
             & ["role"] & ["service"] & ["type"])                 # optional, any of

The "type" property implements a type extension mechansim. Type values are resolved by consulting a profile-specific schema. This allows for a late-binding of semantics. Rationale is that multiple applications can build on this common infrastructure and defer semantic overloading till runtime.

Base Classes Base Properties

1. "Item"
A compound object with either a resource property or a collection property together with qualifying properties.
2. "Nest"
A compound object with both a resource property and a collection property together with qualifying properties.

0a. "collection" ~> [uri]
A container for Item or Nest objects with a specified ordering.
0b. "resource" ~> [uri]
A generic resource. This can be variously a meta-resource, eg a DOI (identified by a URI of scheme "doi:"), a familiar web resource (identified by any URI), or a "literal", ie an inline resource (identified by a URI of scheme "data:").

1. "access" ~> [string]
A label to mark off an access point within a resource hierarchy. Example is ":default" to indicate to a service component the default resource set to return.
2. "detail" ~> [string]
A human readable description of an Item or Nest object.
3. "directive" ~> [string]
A directive which controls a service behaviour. Example is a directive to a service component not to bundle a user-supplied parameter string together with a resolution result set.
4. "label" ~> [string]
A presentational element standing for a resource in a display service.
5. "role" ~> [string]
Indicates the relationship of this resource with another resource.
6. "service" ~> [string]
Indicates a service component.
7. "type" ~> [string]
An extensibility type. The value is the application type (or property) of an item or nest, and the definition is maintained in a schema registered under the application profile.
