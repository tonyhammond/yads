yads: Resource Hierarchy

A generalized DObject resource hierarchy (see diagram below) is built up from Members which are resources with the general properties "access", "detail", etc. The Members are contained within Groups which are aggregated into Containers which themselves can be Members of a Group.

The box below shows how this DObject resource hierarchy is represented in a Generic XML description. This Generic XML description of a DObject can be transformed into an Application XML instance via an intermediate Vanilla XML form. 


#1 - Generic XML: Simple description of DObject

<Group>
  <Container type="tube:line" ordering="unordered">
    <Group>
      <Container type="tube:line-branch" ordering="sequenced">
        <Group>
          <Member type="resource">doi:1014/tube/waterloo</Member> 
          <Member type="type">tube:station</Member> 
          <Member type="access">Rail Link</Member> 
        </Group>
        <Group>
          <Member type="resource">doi:1014/tube/bank</Member> 
          <Member type="type">tube:station</Member> 
        </Group>
      </Container>
    </Group>
  </Container>
</Group>

#2 - Vanilla XML form: Resources are "attributed" with properties (aka types)
 
<Resource type="tube:line" ordering="unordered">
  <Resource type="tube:line-branch" ordering="sequenced">
    <Resource type="tube:station" access="Rail Link">doi:1014/tube/waterloo</Resource> 
    <Resource type="tube:station">doi:1014/tube/bank</Resource> 
  </Resource>
</Resource>

#3 - Application XML instance: Typed resources are named

<tube:line>
  <tube:line-branch>
    <tube:station access="Rail Link">doi:1014/tube/waterloo</tube:station> 
    <tube:station>doi:1014/tube/bank</tube:station> 
  </tube:line-branch>
</tube:line>

The reverse is also true - Application XML instances can also be transformed into a Generic XML description via an intermediate Vanilla XML form. 


#1 - Application XML instance: Typed resources are named

<menu:submenu label="">
  <menu:submenu label="DOI Sites">
    <menu:item label="IDF">http://www.doi.org</menu:item> 
    <menu:item label="CNRI">http://www.cnri.reston.va.us</menu:item> 
  </menu:submenu>
</menu:submenu>

#2 - Vanilla XML form: Resources are "attributed" with properties (aka types)
 
<Resource type="menu:submenu" label="" ordering="sequenced">
  <Resource type="menu:submenu" label="DOI Sites" ordering="sequenced">
    <Resource type="menu:item" label="IDF">http://www.doi.org</Resource> 
    <Resource type="menu:item" label="CNRI">http://www.cnri.reston.va.us</Resource> 
  </Resource>
</Resource>

#3 - Generic XML: Simple description of DObject

<Group>
  <Container type="menu:submenu" label="" ordering="sequenced">
    <Group>
      <Container type="menu:submenu" label="DOI Sites" ordering="sequenced">
        <Group>
          <Member type="resource">http://www.doi.org</Member> 
          <Member type="type">menu:item</Member> 
          <Member type="label">IDF</Member> 
        </Group>
        <Group>
          <Member type="resource">http://www.cnri.reston.va.us</Member> 
          <Member type="type">menu:item</Member>
          <Member type="label">CNRI</Member> 
        </Group>
      </Container>
    </Group>
  </Container>
</Group>
