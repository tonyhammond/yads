yads: Vanilla XML

The Vanilla XML description constitutes a middle ground between an Application and a Generic description. An RDF Resource is effectively promoted to a YADS Resource (Group or Container) and is qualified by RDF Literal properties.

Code from the YADS Java client library to build this intermediate description is as below. 


    // Build a simple Resource and attach two Resources
 
    Resource r = new Resource()
      .addResource(new Resource("http://www.doi.org")
        .addLabel("IDF")
        .addDetail("International DOI Foundation.")
        .addAccess("default")
      )
      .addResource(new Resource("http://www.cnri.reston.va.us")
        .addLabel("CNRI")
        .addDetail("Corporation for National Research Initiatives.")
      );

Contrast this with the same code to build a Generic description. 


    // Build a simple Container and attach two Groups
 
    Container c = new Container()
      .addGroup(new Group()
        .addMember(DObjectTypes.DOI_TYPE_RESOURCE, "http://www.doi.org")
        .addMember(DObjectTypes.DOI_TYPE_LABEL, "IDF")
        .addMember(DObjectTypes.DOI_TYPE_DETAIL, "International DOI Foundation.")
        .addMember(DObjectTypes.DOI_TYPE_ACCESS, "default")
      )
      .addGroup(new Group()
        .addMember(DObjectTypes.DOI_TYPE_RESOURCE,
                     "http://www.cnri.reston.va.us")
        .addMember(DObjectTypes.DOI_TYPE_LABEL, "CNRI")
        .addMember(DObjectTypes.DOI_TYPE_DETAIL,
                     "Corporation for National Research Initiatives.")
      );

Note that the intermediate Vanilla XML form can be used "as is" to model a simple menu hierarchy - see descriptions #1 and #2 below which are essentially identical. Also shown is the recovery to a Generic XML description. 


#1 - Application XML instance: Resources are not typed - just plain data

<data label="Further Info">
  <data label="DOI Sites">
    <data label="IDF">http://www.doi.org</data> 
    <data label="CNRI">http://www.cnri.reston.va.us</data> 
  </data>
</data>

#2 - Vanilla XML form: Resources are "attributed" with properties (aka types)
 
<Resource label="Further Info">
  <Resource label="DOI Sites">
    <Resource label="IDF">http://www.doi.org</Resource> 
    <Resource label="CNRI">http://www.cnri.reston.va.us</Resource> 
  </Resource>
</Resource>

#3 - Generic XML: Simple description of DObject

<Group>
  <Container label=">
    <Group>
      <Container label="DOI Sites">
        <Group>
          <Member type="resource">http://www.doi.org</Member> 
          <Member type="label">IDF</Member> 
        </Group>
        <Group>
          <Member type="resource">http://www.cnri.reston.va.us</Member> 
          <Member type="label">CNRI</Member> 
        </Group>
      </Container>
    </Group>
  </Container>
</Group>
